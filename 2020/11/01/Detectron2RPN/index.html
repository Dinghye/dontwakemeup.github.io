【Detectron2】核心Region Proposal Network

检测器如何连接特征图(feature map)和标注框(ground truth)的位置和大小？让我们来看看RPN--RCNN检测器的核心部件——是如何工作的。

图3为RPN的详细示意图。RPN由神经网络（RPN Head）和非神经网络功能组成。RPN³中的所有计算都在Detectron 2的GPU上进行。

首先，我们来看看处理FPN送来的特征图的RPN头。

1.RPN Head

RPN的神经网络部分很简单。它被称为RPN Head，由标准RPNHead类中定义的三个卷积层组成。

  1. cov (3×3，256->256通道)
  2. objectness logits conv（1×1，256 -> 3 ch）
  3. anchor deltas conv (1×1，256->3×4 ch）

五级（P2～P6）的特征图逐一送入网络。

一级的输出特征图为：

  1. pred_objectness_logits（B，3 ch，Hi，Wi）：对象存在的概率图。

  2. pred_anchor_deltas(B，3×4 ch，Hi，Wi)：与锚的相对箱形。

    其中，B代表批量大小，Hi和Wi对应P2到P6的特征图大小。

它们实际上是什么样子的呢？在图4中，将每一级的objectness logits map叠加在输入图像上。你可以发现，小的物体在P2和P3被检测到，大的物体在P4到P6。这正是特征金字塔网络的目标。多尺度网络可以检测到单尺度检测器无法发现的微小物体。

接下来，我们继续进行锚点生成，这对于将标注框与上面两个输出特征图关联起来是至关重要的。

2. 锚点生成 Anchor Generation

为了将objectness map和 anchor deltas map与标注框连接起来，需要称为"锚（anchor）”的参考框。

2.1 生成cell anchor

在Detectron 2的Base-FPN-RCNN中，Anchor的定义如下：

ANCHOR_GENERATOR.SIZEES列表中的五个元素对应五个级别的特征图（P2到P6）。例如P2（stride=4）有一个anchor，其大小为32

长宽比定义了锚的形状。在上面的例子中，有三种形状：0.5、1.0和2.0。让我们看看实际的锚点（图5）。P2特征图处的三个锚点的长宽比分别为1:2、1:1和2:1，面积与32×32相同。在P3级别的锚是P2锚的两倍。

这些锚在Detectron 2中称为"单元锚(cell anchor)”。(锚生成的代码在这里) 结果我们得到了5个特征图级别的3×5=15个单元锚。

2.2 在网格上放置锚点

接下来我们将单元格锚放在网格上，其大小与预测的特征图相同。

例如我们预测的特征图'P6'的大小为（13，20），步幅为64。在图6中，P6网格的三个锚点放置在（5，5）。在输入图像分辨率中，(5，5)的坐标对应于(320，320)，方块锚的大小为(512，512)。锚被放置在每个网格点上，所以13×20×3=780个锚被生成在P6。
对其他网格进行同样的处理（见图6中P5网格的例子），总共生成255,780个锚。

3. 标注数据准备

在本章中，我们将标注框与生成的锚联系起来。

3.1 计算交叉点-单位（IoU）矩阵

假设我们从一个数据集中加载了两个地面真相（GT）盒子。

我们现在试着从255780个anchor中找出与这两个标注盒子相似的盒子。如何判断一个盒子与另一个盒子是否相似呢？答案是交集大于联合（IoU）计算。在Detectron2中，pairwise_iou函数可以从两个盒子列表中计算出每一对盒子的IoU。在我们的例子中，pairwise_iou的结果是一个大小为(2(GT), 255780(anchors))的矩阵。

3.2 用Matcher检查IoU矩阵

Matcher对IoU矩阵进行检查，所有的锚都被标记为前景、背景或忽略。如图7所示，如果IoU大于预先定义的阈值（通常为0.7），则锚被分配到GT框之一，并标记为前景（'1'）。如果IoU小于另一个阈值(通常为0.3)，则将锚标记为背景('0')，否则忽略('-1')。

在图8中，我们展示了叠加在输入图像上的匹配结果。可以看到，大部分网格点被标注为背景（0），少数网格点被标注为前景（1）和忽略（-1）。

3.3 计算锚点三角

被确定为前景的锚箱与GT箱的形状相似。然而，网络应该学习提出GT盒的确切位置和形状。为了实现这一目标，应该学习四个回归参数：Δx、Δy、Δw和Δh。这些"deltas"的计算方法如图9所示，使用Box2BoxTransform.get_deltas函数。该公式写在Faster-RNN论文中。

结果，我们得到一个名为gt_anchor_deltas的张量，在我们的情况下，它的形状是（255,780，4）。

3.4 重新取样计算损失的boxes

现在我们在特征图的每个网格点上都有objectness_logits和anchor_deltas，我们可以将预测的特征图与之进行比较。

  • 问题：

    图10(左)是每张图片的锚点数量和例子的分类。如你所见，大部分的锚都是背景锚。例如，通常在255,780个锚中，foreground 锚不到100个，被忽略的锚不到1000个，其余的都是背景锚。如果我们继续训练，由于标签不平衡，将会很难学会foreground（这里forground指的目标）。

    通过使用subsample_labels函数对标签进行重新采样，解决不平衡问题。

4. 损失计算

在rpn_losses函数处对预测图和ground-truth map应用了两个损失函数。

4.1 本地化损失 (loss_rpn_loc)

  • l1损失

  • 只在ground-truth objectness=1（前景）的网格点上计算，也就是忽略所有背景网格来计算损失。

4.2 客观性损失(损失_rpn_cls)

  • 二元交叉熵损失。

  • 只在ground-truth objectness=1（前景）或0（背景）的网格点上计算。

5.提案选择

最后，我们按照下面的四个步骤从预测框中选择1000个"区域提案"框。

  1. 将预测的anchor_deltas应用到相应的锚中，这是3-3的反向过程。

  2. 预测框按各特征层次的预测对象性得分排序。

  3. 如图11所示，从图像的每个特征级别⁶中选择前K个得分的方框（由配置参数定义）。例如，从P2的192,000个盒子中选择2000个盒子。对于P6存在少于2000个盒子的地方，则选择所有的盒子。

  4. 非最大抑制(batched_nms)在每个级别独立应用。结果有1,000个得分最高的盒子保留下来。

主要术语翻译和解释：

  • Anchor：锚点（用于定位object的框框的点）
  • Grid：网格（图像中各个像素构成的网络）
  • loU：详细见下篇（一个坑）
  • foreground：前景（相对于背景，即所需的目标）
  • box：箱、盒
  • ground-truth ：标注数据（确定为真实的标注数据）
  • proposal ：提案（即可能的结果）
